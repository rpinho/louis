Ask for the whole board

Where the novel, well-founded targets actually are.

You ask @louis — what are your grade-A leads across all diseases? Give me the whole board.

Louis plots every lead by trust grade (how good a lead) against novelty (how understudied that gene is, in T cells, for that disease). A database sends you to the crowded right. Louis is built for the green corner — high-grade and barely in the literature.

The problem with confident tools

Most AI target-finders hand you a ranked list and a plausible story.

The pattern

They surface a lead and a tidy narrative — and give you no signal for when the story is an artifact of their own pipeline. Confidence is the default. The one thing a bench scientist needs — can I trust this? — is exactly what's missing.

What Louis does

Louis leads with the trust verdict. It overrules its own grade-A pick when the wiring is an artifact, tiers every claim by provenance, and reads what the field is saying before the papers land — then hands you the experiment that would settle it.

One engine, three things it does

Discover leads. Weigh trust. Read the bleeding edge.

Trust

Honesty you can watch

Handed its own flagship RA lead DOT1L, Louis stress-tests it A→C: the RA wiring is a cross-disease-union artifact — only 1 of 77 regulator clusters is ever significant — and the direction is a genuine open question. Then it returns the survivor, HDAC7 for Th17-driven colitis, with the go/no-go experiment.

the product: a discovery tool that kills its own darling — with receipts.

Learns

The whole lab writes back

Every finding is weighed by provenance — a preprint can't decide a grade. Louis reviews itself, and the lab teaches it at the bench: results write back to one shared memory, so questions compound instead of restarting — self-correcting, always human-in-the-loop.

the loop: discover → validate → listen → remember.

Signal

Reads the pre-paper floor

Louis pulls the off-allowlist signal a database can't — what immunologists post on X and Bluesky and drop in conference abstracts — then tiers it by signal-strength: a lead to chase, never evidence for a grade.

Having trouble installing Louis?

Read this — or hand the block below straight to your agent.

A fresh clone is missing two things the repo gitignores: the screen data and the prebuilt skill / plugin. Two commands fix both. (Heads up: the repo's CLAUDE.md says the data ships in-repo — that one line is wrong; it's fetched, as below.)

# from a clean clone — run in order, expect no errors
git clone https://github.com/rpinho/louis && cd louis
python3 -m venv .venv
.venv/bin/pip install -e ".[slack]"

# 1) the screen data is NOT in the repo (gitignored, ~34 MB, public/MIT) — fetch it
.venv/bin/python scripts/download_data.py

# 2) sanity check — expect: "3/3 demo-invariant checks passed"
.venv/bin/python tests/test_demo_invariants.py

# 3) dist/ is gitignored too — build the surface you want
.venv/bin/python scripts/build_skill.py   # -> dist/louis-skill.zip  (Claude Science)
.venv/bin/python scripts/build_mcpb.py    # -> dist/louis.mcpb       (Claude Desktop)
# Claude Code needs neither — approve the repo's .mcp.json and the tools are live

No such file: data/*.csv

missing data

The five screen tables are gitignored, so a clone has none — the sanity test fails, or every query comes back empty. Fix: python scripts/download_data.py pulls the small precomputed tables from the dataset authors' public MIT repo (no auth).

CLAUDE.md's "ships in the repo" line is wrong

No such file: dist/louis-skill.zip

not built yet

The prebuilt skill zip and .mcpb are gitignored — you build them from public sources. Fix: run scripts/build_skill.py (skill) or scripts/build_mcpb.py (desktop plugin) before you upload/install the file.

build first, then upload / install

ModuleNotFoundError: slack_bolt

wrong Python

You're on the system python3, not the venv. Fix: always call .venv/bin/python (never bare python), and install with the Slack extra: pip install -e ".[slack]".

the venv must be .venv in the repo root
